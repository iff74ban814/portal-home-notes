# portal-home-notes

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库内容以静态 HTML 文件为主，不针对任何特定域名或具体网站，适用于日常记录、页面存档或临时展示。

## 项目简介

本仓库主要存放各类独立编写的 HTML 页面，可能包含信息卡片、工具页面、展示页或笔记式页面。每个页面均为自包含的 HTML 文件，不依赖外部数据库或后端服务，方便直接部署或本地查看。

## 目录说明

```
portal-home-notes/
├── index.html            # 入口页面（可选）
├── pages/                # 各独立 HTML 页面存放目录
│   ├── page-xxx.html
│   └── ...
├── assets/               # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md             # 本文件
```

- `pages/`：放置所有独立 HTML 页面，文件命名应简洁明了。
- `assets/`：存放页面共享的静态资源，如样式表、脚本或图片。
- 根目录下的 `index.html` 可作为导航或入口页面（非必须）。

## 页面归档说明

- 每个 HTML 页面应为独立完整内容，包含必要的 `<html>`、`<head>` 和 `<body>` 结构。
- 页面之间不应产生硬依赖，除非明确通过相对路径引用 `assets/` 中的资源。
- 建议在页面头部或注释中注明创建日期、用途或简要说明，方便后续维护。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，若需共享资源则更新 `assets/`。
- 修改页面：直接编辑对应 HTML 文件，注意保持相对路径一致性。
- 删除页面：从 `pages/` 中移除文件，并检查是否有其他页面引用该文件。
- 建议在提交时附带简短说明，描述变更内容或新增页面的用途。
